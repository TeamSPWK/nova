🎯 한 줄 요약

사용자는 NOVA-STATE.md 한 파일만 보면 충분. 슬래시 커맨드 (/nova:plan, /nova:run 등)가 work-item을 자동 관리합니다.

🧑 사람용 cursor: NOVA-STATE.md ← 매일 한 번 보는 곳 🤖 기계용 진실원: .nova/work-items/ ← AI가 자동 관리

📉 왜 필요한가?

❌ Before (v1/v2)

  • NOVA-STATE.md가 wiki처럼 커짐
  • AI가 stale 정보로 같은 작업 반복 추천
  • 사용자가 매번 손으로 정리
  • 진척 측정 불가 (DORA 지표 없음)
  • commit_sha 추적 없음

✅ After (v3)

  • AI가 자동으로 work-item 발급·전이
  • 사람은 marker 외 영역만 손편집
  • .nova/work-items/가 단일 진실원
  • drift 18 룰로 무결성 자동 보장
  • DORA 표준 evidence (commit_sha)

⚡ 5분 시작

A

새 프로젝트

cd ~/my-project
/nova:setup

자동으로 .nova/{schema,work-items,README.md} 생성 + .gitignore marker 블록 추가. 이후 슬래시 커맨드 사용 시 AI가 자동 work-item 발급.

B

기존 v1/v2 STATE 프로젝트 마이그레이션

cd ~/existing-project
/nova:migrate-state

target 옵션 없이! v1/v2 모두 자동 감지 → v3 직행. dry-run 보존율 보고 → 사용자 검수 → apply + drift-check 자동.

C

평상시 사용

/nova:plan "검색 기능 추가"     # → WI-0042 생성 (status=proposed)
/nova:run                       # → 구현 + Evaluator → status=done + commit_sha
/nova:review                    # → review_required=true
/nova:check                     # → drift 18 룰 자동 검증

사용자는 평소 그대로 슬래시 커맨드 사용. 내부에서 work-item 자동 관리. NOVA-STATE.md marker 영역만 봐도 진척 파악 OK.

🔄 5 상태 enum (외울 필요 X)

/nova:plan [ ] ──────────────────▶ proposed │ │ /nova:run 시작 ▼ active ──────▶ done (Evaluator PASS) ▲ │ ───────── │ │ commit_sha 필수 블로커 해소 │ │ review_required=false │ ▼ 원자적 전이 blocked (blocked_reason 필수) │ ▼ superseded ────▶ [폐기]
status의미자동 트리거
proposed계획만 잡힘/nova:plan
active진행 중/nova:run 시작
blocked막힘 (reason 필수)/nova:run 도중 블로커
done완료 + commitEvaluator PASS
superseded폐기/nova:plan으로 대체

원자적 보장: done 전이 시 review_required=false + evidence.commit_sha 동시 set. 부분 갱신 불가능 (SIGINT 안전).

💬 FAQ

Q1. 기존 NOVA-STATE.md 내용 사라지나요?

아니요. 마이그레이션 시 NOVA-STATE.md.v2.bak로 자동 백업. marker 외 영역도 그대로 보존. 변환된 work-item은 .nova/work-items/로 추가되는 것이지 대체가 아닙니다.

Q2. team 공유는?

.nova/work-items/, .nova/schema/, .nova/README.md는 git-tracked. clone 즉시 진실원 복원. .nova/events.jsonl, .nova/local/는 git-ignored (사용자별 로컬).

Q3. 다른 AI 도구(Cursor, Copilot)와 호환?

work-item은 JSON Schema(v3.0) 기반. 다른 도구도 .nova/work-items/*.json 직접 읽기 가능. registry-write.sh API를 따르는 한 어떤 AI든 호환.

Q4. UI/대시보드는?

/nova:status — HTML dashboard 자동 생성 (Phase·Sprint 진행률 + drift 알람). 브라우저로 열기.

Q5. 마이그레이션 실패 시 복구?

mv NOVA-STATE.md NOVA-STATE.md.v3.failed
mv NOVA-STATE.md.v2.bak NOVA-STATE.md
rm -rf .nova/work-items .nova/schema .nova/README.md
# .gitignore의 Nova marker 블록도 제거
