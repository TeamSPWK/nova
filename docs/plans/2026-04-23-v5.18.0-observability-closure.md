# Plan: v5.18.0 — Observability Closure

**작성일**: 2026-04-23
**현재 버전**: v5.17.0
**근거**: v5.17.0 실기 테스트에서 Tier 3.1 구조 갭 발견

## Context

v5.17.0 Tier 3.1(Behavior Learning)이 릴리스됐으나 **분석 대상 데이터가 얕아 의미 있는 결과가 나오지 않음**. 실측:

- `.nova/events.jsonl` 750 이벤트 중 696(93%)이 `session_start`
- 도구 호출(Read/Edit/Bash 등) 이벤트: **0건**
- `orchestration_missing`: 4건 (2026-04-20~23)
- `session_start`가 한 세션 내에서 14회 이상 중복 기록되는 케이스 발견 (예: 2026-04-23 01:25:20~01:25:42 사이 14회)

## Problem

| 문제 | 증거 | 영향 |
|------|------|------|
| **(a) PreToolUse 관측 미구현** | `hooks/hooks.json`에 tool_call 기록 훅 없음 | analyze-observations의 "tool-frequency"가 빈 결과 |
| **(b) session_start 중복 기록** | 20초 내 14회 (extra: `{}` 또는 `{"trigger":"setup-permissions"}`) | events.jsonl 93%가 노이즈 → 분석 정밀도 하락 |
| **(c) orchestration_missing 4건** | 2026-04-20~23 세션 4건, 3~7분 지속인데 orchestration_update 0회 | 메인 에이전트(본인 포함)가 서브에이전트 launch 시 Phase 0 등록 누락 |
| **(d) analyze-observations 스키마 미일치** | tool_call 이벤트 가정하고 작성했으나 실제 스키마엔 없음 | tool-frequency 출력이 session_start/session_end만 나열 |

## Solution

### §1. PreToolUse tool_call 기록 훅 (주 목표)

**`hooks/pre-tool-use-record.sh`** 신규:
- PreToolUse 트리거. `$TOOL_NAME`만 기록 (인자 redact)
- `bash hooks/record-event.sh tool_call '{"tool":"$TOOL_NAME"}'` 호출
- `NOVA_DISABLE_EVENTS=1` 옵트아웃 지원
- 기존 PreToolUse 훅(pre-edit-check, pre-commit-reminder)과 병렬 실행

**`hooks/hooks.json`** 수정:
- PreToolUse 매처 `"*"` 엔트리 추가, pre-tool-use-record.sh 호출
- 기존 Write|Edit / Bash 매처 유지

**Privacy**: `$TOOL_NAME`만 기록. `$TOOL_INPUT`은 기록 금지 (경로/코드 노출 위험).

### §2. session_start debounce

**`hooks/session-start.sh`** 수정:
- `.nova/session.start_epoch` 존재하고 마지막 기록으로부터 5초 이내면 이벤트 기록 스킵
- `extra` 필드에 `trigger` 또는 `reload_count` 추가 (노이즈 식별 가능)

### §3. orchestration_missing 재발 방지

**`hooks/audit-orchestration.sh`** 이미 있음. 강화:
- 3분+ 세션에서 orchestration_update 누락 감지 시 단순 기록이 아니라, `hooks/stop-event.sh`에서 **stderr 경고** 출력 ("Phase 0 등록 누락 — 서브에이전트 launch 시 mcp__plugin_nova_nova__orchestration_update 호출 누락")
- 사용자가 알아챌 수 있게 가시화

### §4. analyze-observations 스키마 일치 + 범용화

**`scripts/analyze-observations.sh`** 수정:
- tool_call 이벤트 누적되면 "도구별 빈도" 실데이터 기반 출력
- session_start 중복은 debounce 후 분석에서 제외 (또는 trigger별 집계)
- failures 패턴에 `orchestration_missing` 포함

### §5. 테스트

`tests/test-scripts.sh` §S8 신설:
- S8.1: hooks.json PreToolUse 매처 "*" 엔트리 존재
- S8.2: pre-tool-use-record.sh 존재 + 실행 권한 + redact 동작 (TOOL_INPUT 기록 안 됨 확인)
- S8.3: session-start.sh debounce — 5초 내 2회 호출 → 1회만 기록
- S8.4: audit-orchestration.sh stop-event 연계 — 경고 stderr 출력
- S8.5: analyze-observations.sh — tool_call 이벤트 fixture로 Top N 검증
