Plan은 2026-04-23 자가 감사로 발견된 Nova 규칙 집행 구조의 **3층 붕괴**를 복구한다.

### 설계 원칙

1. **fail-closed 기본, 명시적 fail-open**: enforcement 레이어는 불확실성이 있으면 차단한다. 우회는 `NOVA_EMERGENCY=1`, `NOVA_DISABLE_EVENTS=1`처럼 사용자가 명시적으로 의도를 밝힌 경우에만 허용한다.
2. **관측 가능성은 집행과 독립**: 이벤트 기록 실패가 집행 자체를 막지 않는다. `record-event` 실패 시 경고 stderr + 훅 본문 로직 계속.
3. **session 경계 투명성**: ledger 매칭은 session_id 불문. Claude Code 재시작, 팀 협업, worktree 분기에서도 작동.
4. **레이어 경계 명시**: Claude Code 훅(Bash 도구 호출 전) vs 네이티브 git 훅(실제 git 실행) 두 레이어는 독립. Nova는 전자만 담당.
5. **forward/backward-compat**: v5.18.x의 analyze-observations가 v5.19.0 신규 이벤트 섞인 jsonl을 읽어도 unknown skip 후 exit 0.
6. **Privacy First**: 파일 경로 자체가 PII이므로 기본적으로 sha256 해시로 기록. 민감 파일은 완전 skip.

## Solution (설계 상세)

### 1. S1.1 pre-commit-reminder.sh 최종 설계 (N1, N3 반영)

**상태 머신 판정 순서**:

```
stdin JSON 수신
  │
  ├─ tty / 미입력 / jq 실패 ──▶ fail-closed (exit 2) *
  │                              * NOVA_DISABLE_EVENTS=1 이면 exit 0
  │
  ▼
COMMAND 추출 + git commit 패턴 매칭
  │
  ├─ 비매칭 ──▶ exit 0 (일반 Bash 호출)
  │
  ▼
[check_evaluator_pass()] 7상태 판정 — 순서 고정
  │
  ├─ NOVA-STATE.md 파일 없음 ──────────▶ MISSING ──▶ fail-closed (안내 메시지)
  │
  ├─ 파일 있음 + grep '^<<<<<<<' 매치 ──▶ CONFLICT ──▶ fail-closed (초월, EMERGENCY 우회 불가) [N1, N3]
  │
  ├─ ## Last Activity 본문 없음 ────────▶ EMPTY ──▶ fail-closed
  │
  ├─ LAST_LINE에 "→ PASS|CONDITIONAL" 없음 ──▶ NO_PASS ──▶ fail-closed (EMERGENCY 우회 가능)
  │
  ├─ TS 파싱 실패 ────────────────────────▶ TIMESTAMP_BROKEN ──▶ fail-closed (EMERGENCY 우회 가능)
  │
  ├─ TS != today ────────────────────────▶ STALE ──▶ fail-closed (EMERGENCY 우회 가능)
  │
  └─ TS == today ────────────────────────▶ PASS ──▶ exit 0
```

**N1 반영**: CONFLICT 체크는 반드시 MISSING 직후, EMPTY 이전. 이유: merge conflict 마커가 awk 섹션 파서를 깨뜨려 STALE/TIMESTAMP_BROKEN로 오분류되는 것 방지.

**N3 반영**: EMERGENCY 우회 가능/불가 분기를 상태별로 명시:

| 상태 | EMERGENCY 우회 | 이유 |
|------|---------------|------|
| PASS | N/A (통과) | — |
| MISSING | 가능 | 초기 세팅 단계 마찰 해소 |
| **CONFLICT** | **불가** | 커밋 자체가 repo 손상 유발. conflict 해결 선결 |
| EMPTY | 가능 | 관측 공백일 뿐 |
| NO_PASS | 가능 | 실제 긴급 프로덕션 수정 시나리오 |
| TIMESTAMP_BROKEN | 가능 | 포맷 오류 복구용 |
| STALE | 가능 | 타임존 간 이슈 등 |

**최종 스크립트 구조**:

```bash
#!/usr/bin/env bash
# Nova Engineering — PreToolUse Hook (Bash) — v5.18.3
# Hard Gate: Evaluator PASS 없이 git commit 차단

# ── NOVA_DISABLE_EVENTS: fail-open 최상위 스위치 (테스트 편의)
[ "${NOVA_DISABLE_EVENTS:-0}" = "1" ] && exit 0

# ── stdin 파싱 (fail-closed 정책)
if [[ ! -t 0 ]]; then
  INPUT_JSON=$(cat 2>/dev/null || true)
  [ -z "$INPUT_JSON" ] && exit 2
  COMMAND=$(echo "$INPUT_JSON" | jq -r '.tool_input.command // empty' 2>/dev/null) || exit 2
else
  exit 0   # tty 수동 실행 (Claude Code 외부) — 차단 안 함
fi

# ── git commit 패턴 필터
echo "$COMMAND" | grep -qE '^\s*git\s+(.*\s+)?commit(\s|$)' || exit 0

# ── EMERGENCY 플래그 감지 (CONFLICT에서는 무시)
EMERGENCY=0
echo "$COMMAND" | grep -q -- "--emergency" && EMERGENCY=1
[ "${NOVA_EMERGENCY:-0}" = "1" ] && EMERGENCY=1

# ── 7상태 판정
check_evaluator_pass() {
  [ ! -f "NOVA-STATE.md" ] && { echo MISSING; return; }
  grep -q '^<<<<<<<' NOVA-STATE.md && { echo CONFLICT; return; }

  LAST_LINE=$(awk '/^## Last Activity/{f=1; next} f && /^- /{print; exit}' NOVA-STATE.md)
  [ -z "$LAST_LINE" ] && { echo EMPTY; return; }

  echo "$LAST_LINE" | grep -qE '→\s*(PASS|CONDITIONAL)' || { echo NO_PASS; return; }

  TS=$(echo "$LAST_LINE" | grep -oE '[0-9]{4}-[0-9]{2}-[0-9]{2}' | head -1)
  [ -z "$TS" ] && { echo TIMESTAMP_BROKEN; return; }

  [ "$TS" = "$(date +%Y-%m-%d)" ] && echo PASS || echo STALE
}

STATE=$(check_evaluator_pass)

# ── CONFLICT는 EMERGENCY 우회 불가 (N3)
if [ "$STATE" = "CONFLICT" ]; then
  cat >&2 << 'EOF'
[Nova Hard Gate] COMMIT BLOCKED — CONFLICT 상태
NOVA-STATE.md에 merge conflict 마커(<<<<<<<)가 있습니다.
NOVA_EMERGENCY 우회 불가. 먼저 conflict를 해결한 뒤 커밋하세요.
EOF
  # commit_blocked 이벤트 기록 (S2)
  bash "${CLAUDE_PLUGIN_ROOT}/hooks/record-event.sh" commit_blocked \
    "$(jq -cn --arg s "$STATE" '{state:$s, emergency:false}')" 2>/dev/null || true
  exit 2
fi

# ── 다른 차단 상태 (EMERGENCY 우회 가능)
if [ "$STATE" != "PASS" ]; then
  if [ "$EMERGENCY" = "1" ]; then
    echo "[Nova Hard Gate] --emergency 우회 — 상태: $STATE" >&2
    bash "${CLAUDE_PLUGIN_ROOT}/hooks/record-event.sh" commit_emergency \
      "$(jq -cn --arg s "$STATE" '{state:$s}')" 2>/dev/null || true
    # 기존 경고 메시지 이어서 출력
  else
    cat >&2 << EOF
[Nova Hard Gate] COMMIT BLOCKED — 상태: $STATE
(상태별 조치 안내 — 기존 로직 유지)
긴급 우회: git commit -m "... --emergency" 또는 NOVA_EMERGENCY=1
EOF
    bash "${CLAUDE_PLUGIN_ROOT}/hooks/record-event.sh" commit_blocked \
      "$(jq -cn --arg s "$STATE" '{state:$s, emergency:false}')" 2>/dev/null || true
    exit 2
  fi
fi

exit 0
```

> record-event 호출은 S2에서 추가. S1 단독 릴리스(v5.18.3)에서는 `2>/dev/null || true`로 safe-default 유지 — record-event.sh가 신규 event_type을 아직 이해 못해도 훅 본문 무영향.

#### 2.1 이벤트 타입 5종 정의

#### 2.2 N2 Privacy 모델 최종 결정

**N2 (Medium, 보안) — v5.19.0 릴리스 블로커** 해소:

**결정**: **옵션 (b) path sha256 해싱을 기본**으로 채택하되, **민감 파일 그룹은 옵션 (a) 완전 skip** 병행.

| 그룹 | 패턴 | 처리 |
|------|------|------|
| **A. Credentials** | `.env`, `.env.*`, `credentials.json`, `*accessKeys*`, `*.pem`, `*.key`, `id_rsa*`, `.aws/credentials`, `.npmrc` | **완전 skip** (files 배열에 미포함) |
| **B. Secret dirs** | `.secret/**`, `.secrets/**`, `secrets/**`, `.ssh/**` | **완전 skip** |
| **C. 일반 파일** | 위 그룹 외 | path sha256 해싱 + 확장자 + size 기록 |

**정당화**:
- Credentials 파일이 staged됐다는 **사실 자체**가 민감 정보(의도적/실수 여부 관계없이). 해시해도 "사용자가 커밋하려 했다"는 메타정보 누출. 완전 skip이 맞다.
- 일반 파일은 ledger 매칭(S3.2)에 필요하므로 sha256으로 기록. 원본 경로는 미기록.
- 사용자 opt-out: `NOVA_PATH_HASH=0` 환경변수 설정 시 path 원본 기록(기본 비활성). 이 경우 사용자가 명시적으로 privacy 완화 선택.

**files[] 스키마 (v5.19.0)**:

```jsonc
{
  "files": [
    {
      "path_hash": "sha256:a1b2c3...",      // 원본 경로 sha256 (기본)
      "path": "src/foo.ts",                 // NOVA_PATH_HASH=0일 때만 존재
      "extension": ".ts",                    // 확장자 (항상 기록)
      "size_bytes": 1234,                    // 파일 크기 (항상 기록)
      "content_sha256": "sha256:d4e5f6..."   // 내용 sha256 (ledger 매칭 키)
    }
  ]
}
```

`content_sha256`은 매칭 키 — S3.2 ledger 조회가 이 필드를 비교.
`path_hash`는 같은 경로의 변경 추적용 (여러 review_pass 간 동일 파일 식별).

#### 2.3 `_privacy-filter.py` 확장 (D3)

기존 API 유지 + 신규 함수:

```python
# hooks/_privacy-filter.py (확장)

SENSITIVE_PATH_PATTERNS = [
    r"^\.env$", r"^\.env\..*$",
    r"^credentials\.json$",
    r".*accessKeys.*",
    r".*\.pem$", r".*\.key$",
    r"^id_rsa.*",
    r"^\.aws/credentials$",
    r"^\.npmrc$",
    r"^\.secret/.*", r"^\.secrets/.*",
    r"^secrets/.*",
    r"^\.ssh/.*",
]

def is_sensitive_path(path: str) -> bool:
    """민감 파일 패턴 매칭. True면 files[] 배열에서 완전 skip."""
    import re
    for pat in SENSITIVE_PATH_PATTERNS:
        if re.match(pat, path):
            return True
    return False

def build_file_record(path: str, content_sha: str, size: int) -> dict | None:
    """민감 파일이면 None 반환 (skip). 일반 파일은 해싱된 레코드 반환."""
    if is_sensitive_path(path):
        return None
    import hashlib, os
    ext = os.path.splitext(path)[1]
    record = {
        "extension": ext,
        "size_bytes": size,
        "content_sha256": f"sha256:{content_sha}",
    }
    if os.environ.get("NOVA_PATH_HASH", "1") == "0":
        record["path"] = path
    else:
        h = hashlib.sha256(path.encode("utf-8")).hexdigest()
        record["path_hash"] = f"sha256:{h}"
    return record
```

# scripts/lib/query-review-ledger.sh
# Usage: query-review-ledger.sh <staged_sha_list_file>
# 출력: stdout = 매칭된 review_pass 이벤트 JSON (최신), 없으면 exit 1

WINDOW_MIN="${NOVA_LEDGER_WINDOW_MIN:-90}"   # N5 반영: 기본 90분

[ "${NOVA_PROFILE:-standard}" = "lean" ] && WINDOW_MIN="${NOVA_LEDGER_WINDOW_MIN:-240}"

CUTOFF_TS=$(date -u -d "$WINDOW_MIN minutes ago" +%s 2>/dev/null || \
            date -u -v-${WINDOW_MIN}M +%s)

# rotation archive 포함 조회 (R9 완화)
for f in .nova/events.jsonl .nova/events.jsonl.{1,2,3,4,5}; do
  [ -f "$f" ] || continue
  jq -c --argjson cutoff "$CUTOFF_TS" \
    'select(.event_type == "review_pass" and (.timestamp_epoch // 0) >= $cutoff)' \
    "$f"
done | while read -r evt; do
  # staged_sha_set ⊆ evt.files[].content_sha256
  LEDGER_SHAS=$(echo "$evt" | jq -r '.files[].content_sha256 // empty')
  if awk -v staged="$(cat "$1")" -v ledger="$LEDGER_SHAS" '
      BEGIN { split(ledger, L, "\n"); for (i in L) H[L[i]] = 1 }
      { if (!($0 in H)) exit 1 }
      END { exit 0 }
  '; then
    echo "$evt"
    exit 0
  fi
done

exit 1

### 4. S4 analyze-observations 설계 (D8)

#### 4.1 self_enforcement_violation 지표

```bash
# scripts/analyze-observations.sh 확장

compute_self_enforcement_violation() {
  local window_min=60   # 고정 60분 (지표 정의 안정성)
  local window_sec=$((window_min * 60))

  # 1. 모든 commit 이벤트 (commit_blocked + commit_emergency + state_sync는 제외)
  # 실제로는 "성공한 commit" 관측이 없으므로, commit_blocked + commit_emergency만 추적
  # Git log와 events.jsonl을 cross-reference하여 '성공한 commit' 도출
  local total_commits
  total_commits=$(git log --since="7 days ago" --format="%H %at" 2>/dev/null | wc -l | tr -d ' ')

  # 2. 1시간 내 review_pass 이벤트 있는 commit
  local covered=0
  while read -r commit_sha commit_ts; do
    local cutoff=$((commit_ts - window_sec))
    if jq -e --argjson c "$cutoff" --argjson t "$commit_ts" \
         'select(.event_type == "review_pass" and
                 (.timestamp_epoch // 0) >= $c and
                 (.timestamp_epoch // 0) <= $t)' \
         .nova/events.jsonl .nova/events.jsonl.{1,2}* 2>/dev/null | head -1 > /dev/null; then
      covered=$((covered + 1))
    fi
  done < <(git log --since="7 days ago" --format="%H %at")

  local violated=$((total_commits - covered))
  echo "{\"total\":$total_commits, \"covered\":$covered, \"violated\":$violated}"
}
```

**지표 분모/분자 정의 (불변)**:
- 분모: 최근 7일간 git commit 수
- 분자: 분모 중 `commit_ts - 60분 ~ commit_ts` 창에 review_pass 이벤트가 **없는** commit 수
- 위반율 = 분자 / 분모 (분모 0이면 N/A)
