제안 방향 (옵션 D):

## 검토 의견

### 1. 방향성 평가: 합리적이나 한 가지 전제 확인 필요

옵션 D는 Karpathy 원칙을 자기 자신에게 적용한 메타적으로 일관된 선택입니다. **단, (a)가 실제로 "2파일"로 완결되는지 검증이 선행**되어야 합니다. criterion 추가가 evaluator 호출 체인이나 스코어링 집계 로직까지 파급되면 범위가 확장됩니다. 이것부터 확인하세요.

---

### 2. 빠진 관점

**Upgrade 모드 지연 리스크가 가장 큽니다.** 숫자 불일치(17↔32)와 자연어 의존 탐지는 현재도 잘못된 신뢰를 생성 중입니다. "별건 분리"는 맞지만, **이슈 티켓/TODO로 명시적 기록 없이 미루면 영구 부채**가 됩니다. 분리 결정 자체를 커밋 메시지나 문서에 남기세요.

**기존 사용자 영향:** (c) 숫자 정정은 breaking change가 아니지만, 사용자가 "17개 점검했다"는 기록을 남겼다면 혼란이 생깁니다. CHANGELOG 한 줄이면 충분합니다.

---

### 3. 템플릿 경량화: 조건부 안전

**핵심 문제:** 플러그인 미설치 사용자는 규칙을 못 받습니다.

권장 처리:
```
# Claude 동작 규칙
> Nova 플러그인 설치 시 자동 주입됩니다.  
> 미설치 환경: [rules-standalone.md] 내용을 이 섹션에 수동 복사하세요.
```

즉, **참조 링크 + 폴백 경로를 명시**하면 제거해도 안전합니다. "한 줄 참조"만 남기고 폴백 없이 삭제하면 미설치 사용자에게는 무언의 기능 손실이 됩니다.
