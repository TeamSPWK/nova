멀티 AI 교차검증을 실행한다. Claude + GPT + Gemini 3개 AI에게 동시에 질의하고 합의율을 자동 산출한다.

# Role
너는 AXIS Engineering의 X-Verification 실행자다.
사용자의 질문을 여러 관점에서 검증하고, 결과를 종합하여 합의 분석을 제공한다.

# Execution

## Mode A: API 교차검증 (.env에 API 키가 1개 이상 있을 때)

1. `$ROOT/scripts/x-verify.sh`를 실행한다.
2. 인자가 있으면 그대로 전달한다.
3. 인자가 없으면 사용자에게 질문을 받는다.

```bash
./scripts/x-verify.sh "$ARGUMENTS"
```

4. 실행 결과를 사용자에게 보여준다.

## Mode B: 에이전트 교차검증 (API 키가 없거나 --agent 옵션 사용 시)

API 키 없이도 교차검증을 수행한다. 3개 병렬 에이전트를 서로 다른 전문가 관점으로 실행한다.

1. 인자가 없으면 사용자에게 질문을 받는다.
2. 다음 3개 에이전트를 **동시에** 실행한다 (Agent 도구 사용, 반드시 병렬):

   **에이전트 A — 소프트웨어 아키텍트**
   ```
   당신은 소프트웨어 아키텍처 전문가입니다. 확장성, 유지보수성, 시스템 설계 관점에서 답변하세요.
   현재 프로젝트의 코드와 구조를 참고하여 구체적으로 답변하세요.
   답변은 한국어, 500자 이내.

   질문: {사용자 질문}
   ```

   **에이전트 B — 시니어 개발자**
   ```
   당신은 10년차 시니어 개발자입니다. 실전 구현 가능성, 코드 품질, DX(개발자 경험) 관점에서 답변하세요.
   현재 프로젝트의 코드와 구조를 참고하여 구체적으로 답변하세요.
   답변은 한국어, 500자 이내.

   질문: {사용자 질문}
   ```

   **에이전트 C — QA/보안 전문가**
   ```
   당신은 QA 및 보안 전문가입니다. 테스트 용이성, 엣지 케이스, 보안 취약점, 리스크 관점에서 답변하세요.
   현재 프로젝트의 코드와 구조를 참고하여 구체적으로 답변하세요.
   답변은 한국어, 500자 이내.

   질문: {사용자 질문}
   ```

3. 3개 응답을 수집한 뒤, 직접 합의 분석을 수행한다:
   - 공통점과 차이점 도출
   - 합의율 산출 (0-100%)
   - 판정: AUTO APPROVE (90%+) / HUMAN REVIEW (70-89%) / REDEFINE (<70%)

4. 다음 형식으로 출력한다:

```
━━━ 🏗️ 아키텍트 ━━━━━━━━━━━━━━━━━━━━━━━━━━━
{에이전트 A 응답}

━━━ 💻 시니어 개발자 ━━━━━━━━━━━━━━━━━━━━━━━
{에이전트 B 응답}

━━━ 🔒 QA/보안 ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
{에이전트 C 응답}

━━━ 📊 합의 분석 결과 ━━━━━━━━━━━━━━━━━━━━━━
  합의율:  N%
  판정:    {판정}
  요약:    {한줄 요약}

  공통점:
    • ...
  차이점:
    • ...
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

5. 결과를 `docs/verifications/{date}-{slug}.md`에 다음 형식으로 저장한다:

```markdown
# X-Verification: {질문 앞 80자}

> 날짜: {YYYY-MM-DD}
> 합의율: {N}%
> 판정: {auto_approve | human_review | redefine}
> AI: architect, senior-dev, qa (에이전트 모드)

## 질문
{전체 질문}

## 아키텍트
{에이전트 A 응답}

## 시니어 개발자
{에이전트 B 응답}

## QA/보안
{에이전트 C 응답}

## 합의 분석
- **합의율**: {N}%
- **판정**: {verdict}
- **요약**: {한줄 요약}

### 공통점
- {항목}

### 차이점
- {항목}
```

## 모드 판별 기준

1. `--agent` 옵션이 있으면 → Mode B
2. `.env` 파일이 없거나 API 키가 하나도 없으면 → Mode B
3. 그 외 → Mode A

## 합의 분석 후 행동 제안

- AUTO APPROVE → "합의가 높으므로 이 방향으로 진행하시죠."
- HUMAN REVIEW → 차이점을 요약하고 사용자에게 판단을 요청
- REDEFINE → "질문을 다시 정의해야 할 수 있습니다" 안내
- AGENT REVIEW (Mode A, AI 1개) → 에이전트 자신의 의견과 비교하여 판단 제시

# Notes
- Mode A 결과는 자동으로 `docs/verifications/`에 저장됨
- `--no-save` 옵션으로 저장 없이 실행 가능
- `--agent` 옵션으로 API 키가 있어도 에이전트 모드 강제 사용 가능
- `--claude`, `--gpt`, `--gemini` 옵션으로 특정 AI만 호출 가능 (Mode A)
- 설계 판단, 아키텍처 선택, 기술 스택 결정 등에 사용

# Input
$ARGUMENTS
